## 第1章 项目概览

### 1.1 项目定位

`d:\Unity\LockstepDemo\CodeMode` 是一个**最小化可运行的 ET 框架学习项目**，在原 CodeMode Demo（演示 asmdef/asmref 切换机制）基础上，新增了：

1. **10 个 ET 基础设施包**（cn.etetet.core / loader / hybridclr / memorypack 等）
2. **一个完整的角色控制模块**（移动 + 跳跃 + 动画切换）
3. **ET 原版 Loader 运行时加载方案**（GameInit.cs + ET.Entry.Start + MainSceneViewInit）

通过本项目，你可以学到：
- ET 框架的 **Entity-Component-System (ECS)** 设计思想
- ET **四层架构**（Model / Hotfix / ModelView / HotfixView）的代码分层
- **World / Fiber / Scene** 等 ET 核心概念如何串联
- **ET 原版 Loader 启动流程**（CodeLoader → ET.Entry.Start → 业务初始化）
- 如何编写符合 ET 规范的**热更代码**（[EntitySystemOf] / [EntitySystem] / [FriendOf]）

### 1.2 与原 CodeMode Demo 的差异

| 维度 | 原 CodeMode Demo | 当前项目 |
| --- | --- | --- |
| ET 包依赖 | 0 个 | 10 个（含 core/loader/hybridclr/memorypack 等） |
| 业务代码 | HelloWorld 示例 | 角色控制模块（Model/Hotfix/ModelView/HotfixView 四层完整） |
| Unity 入口 | 无 | GameInit.cs（基于 ET 原版 Init.cs）+ ET.Entry.Start（Hotfix 层业务入口） |
| 启动方式 | 无 | ET 原版 Loader 方案（CodeLoader 反射调用 ET.Entry.Start） |
| 运行能力 | 仅演示编译切换 | 可在 Unity Play 后用 WASD 控制角色移动、空格跳跃 |
| HotfixView.asmdef | noEngineReferences: true | noEngineReferences: false（与 ET 原版一致，允许 HotfixView 用 UnityEngine） |

### 2.2 每个包的功能与本项目的使用方式

#### ① cn.etetet.core（ET 框架核心）

**功能**：ET 框架的核心运行时，实现了：
- **Entity-Component-System (ECS)**：Entity 基类、Component 挂载机制、System 反射注册
- **Fiber（纤程）**：轻量级线程调度单元，每个 Fiber 拥有独立 MainThread 和 Scene
- **World**：全局单例容器，管理所有 ISingleton 实例
- **Singleton 模式**：`Singleton<T>` 基类，所有全局服务（Options/TimeInfo/IdGenerater/FiberManager 等）继承它
- **网络通信**：AChannel / KChannel / TChannel 等网络通道（本项目未使用）
- **ObjectPool**：Entity 实例复用池
- **CodeTypes**：程序集扫描与系统注册入口

**本项目使用**：
- `World.Instance.AddSingleton<T>()` 注册所有单例
- `FiberManager.Instance.CreateMainFiber()` 创建主 Fiber
- `Entity.AddComponent<K>()` / `GetComponent<K>()` / `GetParent<T>()` 操作 ECS
- `[ComponentOf]` / `[EntitySystemOf]` / `[EntitySystem]` / `[FriendOf]` 特性来自此包

**关键源码位置**：
- [ET/Packages/cn.etetet.core/Scripts/Core/Share/World/World.cs](../../ET/Packages/cn.etetet.core/Scripts/Core/Share/World/World.cs)
- [ET/Packages/cn.etetet.core/Scripts/Core/Share/World/Fiber/FiberManager.cs](../../ET/Packages/cn.etetet.core/Scripts/Core/Share/World/Fiber/FiberManager.cs)
- [ET/Packages/cn.etetet.core/Scripts/Core/Share/Entity/Entity.cs](../../ET/Packages/cn.etetet.core/Scripts/Core/Share/Entity/Entity.cs)

#### ② cn.etetet.loader（加载器）

**功能**：ET 的代码加载器，用于在运行时加载 Model / ModelView / Hotfix / HotfixView 程序集。完整 ET 项目的客户端启动流程：
1. Unity 启动 → 加载 `ET.Loader.dll`
2. Loader 从磁盘/资源包读取 `ET.Hotfix.dll` / `ET.HotfixView.dll`（HybridCLR 编译产物）
3. 调用 `CodeLoader.Start()` 反射调用 `ET.Entry.Start()` 开始游戏

**本项目使用**（已采用 ET 原版 Loader 方案）：
- [Assets/Scripts/GameInit.cs](../Assets/Scripts/GameInit.cs) 基于 ET 原版 `Init.cs`，调用 `CodeLoader.Start()`
- [Hotfix/Share/Entry.cs](../Packages/cn.codemode.helloworld/Scripts/Hotfix/Share/Entry.cs) 实现 `ET.Entry.Start()` 业务入口
- `CodeLoader` 在编辑器模式下从 AppDomain 获取已加载程序集，运行时模式下从资源包加载 dll
- 详细原理请参考 [ET Loader说明文档.md](./ET Loader说明文档.md)

**关键源码位置**：
- [ET/Packages/cn.etetet.loader/Scripts/Loader/Client/Init.cs](../../ET/Packages/cn.etetet.loader/Scripts/Loader/Client/Init.cs) - ET 原版入口
- [ET/Packages/cn.etetet.loader/Scripts/Loader/Client/CodeLoader.cs](../../ET/Packages/cn.etetet.loader/Scripts/Loader/Client/CodeLoader.cs) - 程序集加载器

#### ③ cn.etetet.memorypack（高性能序列化）

**功能**：源自 [Cysharp/MemoryPack](https://github.com/Cysharp/MemoryPack)，ET 集成版。提供极高性能的二进制序列化，主要用于：
- **帧同步快照**：Lockstep 模式下每帧保存 LSWorld 状态
- **存档/读档**：游戏状态持久化
- **进程内缓存**：高性能对象序列化

**与 Proto 的区别**：
| 维度 | MemoryPack | Proto (protobuf-net) |
| --- | --- | --- |
| 用途 | 进程内序列化（快照、缓存） | 跨网络 RPC 通信 |
| 性能 | 极高（零字符串编码） | 高 |
| 跨语言 | 否（C# 专用） | 是（多语言支持） |
| 典型场景 | 帧同步 LSWorld.SaveLSWorld | 客户端 ↔ 服务端消息收发 |

**本项目使用**：未直接使用（角色控制不需要序列化），仅为 `cn.etetet.core` 的传递依赖。

#### ④ cn.etetet.proto（Proto 协议工具）

**功能**：Proto 协议定义与 `proto2cs` 工具，把 `.proto` 文件导出成 C# 代码。完整 ET 项目中，所有网络消息（如 `C2G_LoginRequest`、`G2C_LoginResponse`）都在 Proto 中定义，自动生成 C# 类。

**本项目使用**：未直接使用（角色控制不涉及网络消息），仅为 `cn.etetet.startconfig` 的传递依赖。

#### ⑤ cn.etetet.sourcegenerator（C# 源代码生成器）

**功能**：ET 的 Roslyn SourceGenerator，编译期扫描代码中的特性（如 `[EntitySystemOf]`、`[FriendOf]`、`[Event]`），自动生成辅助代码：
- 为每个 `[EntitySystemOf]` 类生成系统注册代码
- 为每个 `[FriendOf]` 生成字段访问代理（绕过 private 限制）
- 为 `[Event]` 自动注册事件处理器

**本项目使用**：编译期自动运行，无需手动调用。我们写的 `[EntitySystemOf(typeof(JumpComponent))]` 等特性就是由它处理。

**效果示例**：
```csharp
// 你写的代码
[EntitySystemOf(typeof(JumpComponent))]
public static partial class JumpSystem { ... }

// SourceGenerator 自动生成的代码（不可见，在编译产物中）
partial class JumpSystem : IEntitySystemRegister {
    public void Register() {
        EntitySystemSingleton.Instance.Register<JumpComponent>(this);
        // ...
    }
}
```

#### ⑥ cn.etetet.config（配置工具）

**功能**：内部保留 `ExcelExporter` 工具，用于把 Excel 配置表导出成 C# 代码和二进制数据。完整 ET 项目中，数值表（伤害、经验、物品属性）通过此工具生成。

**本项目使用**：未直接使用，仅为 `cn.etetet.startconfig` 的传递依赖。

#### ⑦ cn.etetet.startconfig（服务器起服配置）

**功能**：服务器启动配置定义。完整 ET 项目中，服务端进程启动时读取 StartConfig JSON 文件，决定：

**本项目使用**：未直接使用（本项目无服务端），仅为 `cn.etetet.loader` 的传递依赖。

#### ⑧ cn.etetet.yooassets（资源管理）

**功能**：集成 [YooAsset](https://github.com/tuyoogame/YooAsset) 资源管理系统，提供：

- 资源版本管理

**本项目使用**：未直接使用（角色控制不需热更资源），仅为 `cn.etetet.loader` 的传递依赖。

#### ⑨ com.etetet.init（ET 初始化）

**功能**：ET 初始化相关代码，包含 `CodeModeChangeHelper.cs`（CodeMode 切换工具的 ET 原版实现）。完整 ET 项目中，Unity 菜单 `ET/Init/CodeMode` 通过 `dotnet run` 调用此包中的控制台程序。

## 第4章 6 个 asmdef 程序集详解

主包 `cn.codemode.demo` 在 `Runtime/` 下定义了 6 个 asmdef（程序集定义），业务包 `cn.codemode.helloworld` 的代码通过 `asmref` 文件归属到对应程序集。

### 4.1 程序集总览表

| asmdef 文件 | 程序集名 | noEngineReferences | references | 职责 | 可热更 |
| --- | --- | --- | --- | --- | --- |
| `Runtime/Model/ET.Model.asmdef` | `ET.Model` | false | [ET.Model] | 数据模型层 | 否 |
| `Runtime/Hotfix/ET.Hotfix.asmdef` | `ET.Hotfix` | **true** | [ET.Hotfix, ET.Model] | 业务逻辑层 | **是** |
| `Runtime/ModelView/ET.ModelView.asmdef` | `ET.ModelView` | false | [ET.ModelView, ET.Model] | 表现层数据 | 否 |
| `Runtime/HotfixView/ET.HotfixView.asmdef` | `ET.HotfixView` | **false** | [ET.HotfixView, ET.ModelView, ET.Hotfix] | 表现层逻辑 | **是** |
| `Runtime/Config/ET.Config.asmdef` | `ET.Config` | true | [] | 配置层 | 否 |
| `Runtime/Editor/ET.Editor.asmdef` | `ET.Editor` | false | [ET.Model] | 编辑器扩展 | 否 |

> ⚠️ **注意 HotfixView 的 noEngineReferences 是 false**！这与 ET 原版 [HotfixView.asmdef](../../ET/Packages/cn.etetet.statesync/Runtime/HotfixView/ET.HotfixView.asmdef) 一致。原因见第5章详解。

### 4.2 程序集引用关系图

```
                    │   ET.Model   │  ← 数据地基（可引用 UnityEngine）
                    └──────┬───────┘
                           │
              ┌────────────┴─────────────┐
              ▼                          ▼
       ┌──────────────┐          ┌──────────────┐
       │  ET.Hotfix   │          │ ET.ModelView │  ← 表现层数据
       │ (不引用 Unity)│          │ (可引用 Unity)│
       └──────┬───────┘          └──────┬───────┘
              │                         │
              └────────────┬────────────┘

                   ┌────────────────┐
                   │ ET.HotfixView  │  ← 表现层逻辑（可引用 UnityEngine）
                   └────────────────┘
```

### 4.3 noEngineReferences 字段含义

`noEngineReferences` 是 Unity asmdef 的字段：
- `false`：自动引用 UnityEngine.dll 和 UnityEditor.dll，**可以 `using UnityEngine;`**
- `true`：**不自动引用** UnityEngine.dll，但**不是禁止引用**（仍可通过 `overrideReferences + precompiledReferences` 手动添加）

为什么 Hotfix / Config 设 true？
- 这两层需要被**服务端纯 .NET 进程**加载（Linux 上 `dotnet run` 启动，无 Unity 引擎）
- 需要 HybridCLR 热更（热更 dll 与 Unity 主工程解耦）

为什么 HotfixView 设 false？
- HotfixView 是**客户端表现层逻辑**，必须操作 Transform、CharacterController、Animator 等 UnityEngine 类型
- 服务端**不会加载 HotfixView**（服务端只有 Server/Share 目录代码）
- 与 ET 原版保持一致

## 第5章 ET 四层架构与 noEngineReferences 设计

### 5.1 四层架构对照表

| 层 | 程序集 | 职责 | 引用 UnityEngine | 可热更 | 跨端共享 |
| --- | --- | --- | --- | --- | --- |
| Model | ET.Model | Entity / Component / 数据结构 | 可（false） | 否 | C/S 共享 |
| Hotfix | ET.Hotfix | System / 业务逻辑方法 | 否（true） | **是** | C/S 共享 |
| ModelView | ET.ModelView | 客户端表现层组件（Transform/Animator 引用） | 是（false） | 否 | 仅客户端 |
| HotfixView | ET.HotfixView | 表现层 System（动画切换/UI刷新） | 是（false） | **是** | 仅客户端 |

### 5.2 为什么 Model 不可热更？

**Model 是地基**，所有人（Hotfix / ModelView / HotfixView）都引用它。如果 Model 改了字段：
- Hotfix dll 重新加载后，旧 Hotfix 找不到新字段会报错
- ModelView / HotfixView 编译时也需要 Model 类型签名一致

因此 Model 修改 = **发布新版客户端**，不能热更。新增字段必须保证向后兼容。

### 5.3 为什么 Hotfix 和 HotfixView 可以热更？

- Hotfix / HotfixView 通过 **HybridCLR** 在运行时加载新 dll
- 修改业务逻辑（System 方法体）只需下发新 Hotfix dll，无需重装客户端
- 这是 ET 热更的核心机制

### 5.4 为什么 ModelView 和 HotfixView 分开？

万一都改了要热更不是更好吗？因为 **数据（ModelView）和逻辑（HotfixView）的变更频率差异巨大**：
- ModelView 改一次（如新增 UI 引用字段）需要全客户端配合
- HotfixView 几乎天天改（动画参数调整、UI 刷新策略）

把它们分开，HotfixView 高频热更，ModelView 低频随版本发布，**热更包体小、加载快**。

### 5.5 Hotfix 层不能用 UnityEngine 的实战约束

在 [JumpSystem.cs](../Packages/cn.codemode.helloworld/Scripts/Hotfix/Share/JumpSystem.cs) 中可以看到：

```csharp
namespace ET  // 注意：不 using UnityEngine
{
    [EntitySystemOf(typeof(JumpComponent))]
    public static partial class JumpSystem
    {
        [EntitySystem]
        private static void Update(this JumpComponent self)
        {
            // ❌ 不能用 Time.deltaTime（UnityEngine）
            // ✅ 用固定步长（与逻辑帧率一致）
            float dt = 0.02f;

            // ❌ 不能用 Vector3
            // ✅ 用 float 计算垂直速度
            self.VerticalVelocity -= self.Gravity * dt;
            self.DeltaY = self.VerticalVelocity * dt;
        }
    }
}
```

而 [MoveSystem.cs](../Packages/cn.codemode.helloworld/Scripts/HotfixView/Client/MoveSystem.cs) 在 HotfixView 层就可以用 UnityEngine：

```csharp
using UnityEngine;  // ✅ HotfixView 允许

namespace ET.Client
{
    [EntitySystemOf(typeof(MoveComponent))]
    public static partial class MoveSystem
    {
        public static void UpdateMove(this MoveComponent self, InputRequest input)
        {
            float dt = Time.deltaTime;  // ✅ 可以用
            self.TargetDirection = new Vector3(input.Horizontal, 0f, input.Vertical);  // ✅ Vector3
            self.DeltaPosition = self.CurrentDirection * self.Speed * dt;
        }
    }
}
```

### 5.6 设计原则总结

| 代码需要 | 放在哪一层 |
| --- | --- |
| 纯数值计算（int/float/bool） | Hotfix |
| 涉及 UnityEngine.Vector3 / Quaternion | HotfixView 或 ModelView |
| 涉及 Transform / Rigidbody / CharacterController 操作 | HotfixView |
| 涉及 Input / Camera / Physics | HotfixView |
| 数据字段定义（不操作 Unity） | Model 或 ModelView |

```csharp
[EntitySystemOf(typeof(JumpComponent))]
[FriendOf(typeof(JumpComponent))]
public static partial class JumpSystem
{
    [EntitySystem]
    private static void Awake(this JumpComponent self) { /* 初始化字段 */ }

    [EntitySystem]
    private static void Update(this JumpComponent self)  // 每帧调用（因 IUpdate）
    {
        float dt = 0.02f;  // 固定步长，不能用 Time.deltaTime

        // 跳跃触发
        if (self.IsGrounded && self.JumpRequested)
        {
            self.VerticalVelocity = self.JumpForce;
            self.IsGrounded = false;
        }
        self.JumpRequested = false;

        // 重力
        if (!self.IsGrounded)
            self.VerticalVelocity -= self.Gravity * dt;

        // 本帧垂直位移
        self.DeltaY = self.VerticalVelocity * dt;

        // 注意：地面检测不在这里做！需要 CharacterController.isGrounded（UnityEngine）
        // 地面检测在 PlayerViewSystem.Update 中做，回写 IsGrounded
    }
}
```

#### [MoveSystem.cs](../Packages/cn.codemode.helloworld/Scripts/HotfixView/Client/MoveSystem.cs)

```csharp
[EntitySystemOf(typeof(MoveComponent))]
public static partial class MoveSystem
{
    public static void UpdateMove(this MoveComponent self, InputRequest input)
    {
        float dt = Time.deltaTime;  // ✅ HotfixView 可以用
        self.TargetDirection = new Vector3(input.Horizontal, 0f, input.Vertical);
        if (self.TargetDirection.sqrMagnitude > 1f) self.TargetDirection.Normalize();

        // 平滑过渡：当前方向 → 目标方向
        self.CurrentDirection = Vector3.MoveTowards(
            self.CurrentDirection, self.TargetDirection, self.Acceleration * dt);

        self.DeltaPosition = self.CurrentDirection * self.Speed * dt;
        self.IsMoving = self.CurrentDirection.sqrMagnitude > 0.01f;
    }
}
```

#### [PlayerViewSystem.cs](../Packages/cn.codemode.helloworld/Scripts/HotfixView/Client/PlayerViewSystem.cs) ⭐ 核心驱动

```csharp
[EntitySystemOf(typeof(PlayerViewComponent))]
public static partial class PlayerViewSystem
{
    [EntitySystem]
    private static void Update(this PlayerViewComponent self)
    {
        if (self.PlayerTransform == null || self.Controller == null) return;

        PlayerComponent player = self.GetParent<PlayerComponent>();

        // 1. 采集输入
        InputComponent inputComp = player.GetComponent<InputComponent>();
        inputComp?.CollectInput();

        // 2. 更新移动
        MoveComponent moveComp = player.GetComponent<MoveComponent>();
        moveComp?.UpdateMove(inputComp?.Input ?? default);

        // 3. 拼接位移向量（水平 + 垂直）
        Vector3 delta = moveComp?.DeltaPosition ?? Vector3.zero;
        JumpComponent jumpComp = player.GetComponent<JumpComponent>();
        if (jumpComp != null)
        {
            // 传递跳跃请求（Hotfix 层的 JumpSystem 会消费）
            if (inputComp != null && inputComp.Input.JumpRequested)
            {
                jumpComp.JumpRequested = true;
                // struct 属性读-改-写回
                InputRequest consumed = inputComp.Input;
                consumed.JumpRequested = false;
                inputComp.Input = consumed;
            }
            delta.y = jumpComp.DeltaY;  // 应用垂直位移
        }

        // 4. 通过 CharacterController 应用位移（带碰撞）
        CollisionFlags flags = self.Controller.Move(delta);

        // 5. 地面检测（回写 JumpComponent.IsGrounded）
        if (jumpComp != null)
        {
            bool grounded = self.Controller.isGrounded;
            if (grounded && !jumpComp.IsGrounded && jumpComp.VerticalVelocity < 0)
            {
                jumpComp.VerticalVelocity = 0f;
                jumpComp.IsGrounded = true;
            }
        }

        // 6. 朝向移动方向
        if (moveComp != null && moveComp.IsMoving)
        {
            Quaternion targetRot = Quaternion.LookRotation(moveComp.CurrentDirection);
            self.PlayerTransform.rotation = Quaternion.Slerp(
                self.PlayerTransform.rotation, targetRot, 10f * Time.deltaTime);
        }
    }
}
```

**这是整个角色控制的核心**：采集输入 → 触发逻辑系统 → 应用位移 → 检测地面 → 旋转朝向。

#### [PlayerAnimatorSystem.cs](../Packages/cn.codemode.helloworld/Scripts/HotfixView/Client/PlayerAnimatorSystem.cs)

```csharp
[EntitySystemOf(typeof(PlayerAnimatorComponent))]
public static partial class PlayerAnimatorSystem
{
    [EntitySystem]
    private static void Update(this PlayerAnimatorComponent self)
    {
        if (self.Animator == null) return;

        PlayerComponent player = self.GetParent<PlayerComponent>();
        MoveComponent moveComp = player.GetComponent<MoveComponent>();
        JumpComponent jumpComp = player.GetComponent<JumpComponent>();

        PlayerAnimState newState;
        if (jumpComp != null && !jumpComp.IsGrounded)
            newState = jumpComp.VerticalVelocity > 0 ? PlayerAnimState.Jump : PlayerAnimState.Fall;
        else if (moveComp != null && moveComp.IsMoving)
            newState = PlayerAnimState.Move;
        else
            newState = PlayerAnimState.Idle;

        if (newState != self.CurrentState)
        {
            self.CurrentState = newState;
            self.Animator.SetInteger("State", (int)newState);
        }
        self.Animator.SetFloat("Speed", moveComp?.CurrentDirection.magnitude ?? 0f);
    }
}
```

### 7.2 每帧 Update 的系统调用顺序

ET 的 `FiberManager.Update()` 会自动调用所有注册的 IUpdate 系统：

```
FiberManager.Update()
        │
        ▼
EntitySystemSingleton.Instance.Update()
        │
        ├─► JumpSystem.Update (Hotfix)              ← 跳跃物理计算
        │   ├─ 处理跳跃触发
        │   ├─ 重力递减 VerticalVelocity
        │   └─ 输出 DeltaY
        │
        ├─► PlayerViewSystem.Update (HotfixView)    ← 核心驱动
        │   ├─ InputComponent.CollectInput()        ← 采集 WASD/空格
        │   ├─ MoveComponent.UpdateMove()           ← 平滑移动计算
        │   ├─ JumpComponent.JumpRequested 传递     ← 触发下一帧跳跃
        │   ├─ CharacterController.Move(delta)      ← 应用位移（带碰撞）
        │   ├─ 地面检测 → 回写 IsGrounded
        │   └─ Transform.rotation 朝向移动方向
        │
        └─► PlayerAnimatorSystem.Update (HotfixView) ← 动画切换
            ├─ 根据 IsGrounded / VerticalVelocity 决定 Jump/Fall
            ├─ 根据 IsMoving 决定 Move/Idle
            └─ Animator.SetInteger("State", ...)
```

### 7.3 跳跃动作的完整数据流

以"按空格跳跃"为例，追踪数据如何在四层中流转：

```
1. 用户按空格
        │
        ▼
2. PlayerViewSystem.Update 调用 inputComp.CollectInput()
   → InputSystem.CollectInput 中：
     Input.GetKeyDown(KeyCode.Space) → input.JumpRequested = true
   → 写入 InputComponent.Input (ModelView 层)
        │
        ▼
3. PlayerViewSystem.Update 下一行：
   if (inputComp.Input.JumpRequested) {
       jumpComp.JumpRequested = true;   ← 传递给 Hotfix 层组件
       inputComp.Input.JumpRequested = false;  ← 消费 InputComponent 的请求
   }
        │
        ▼
4. (下一帧) JumpSystem.Update 被自动调用：
   if (self.IsGrounded && self.JumpRequested) {
       self.VerticalVelocity = self.JumpForce;  // 6f
       self.IsGrounded = false;
   }
   self.JumpRequested = false;
   self.VerticalVelocity -= self.Gravity * dt;  // 15 * 0.02 = 0.3
   self.DeltaY = self.VerticalVelocity * dt;    // 6*0.02 = 0.12
        │
        ▼
5. PlayerViewSystem.Update 读取 jumpComp.DeltaY：
   delta.y = jumpComp.DeltaY;
   self.Controller.Move(delta);  ← CharacterController 应用垂直位移
        │
        ▼
6. 角色上升，脱离地面：
   self.Controller.isGrounded = false
        │
        ▼
7. PlayerAnimatorSystem.Update 检测到 !IsGrounded 且 VerticalVelocity > 0：
   newState = PlayerAnimState.Jump
   self.Animator.SetInteger("State", 2);  ← 播放跳跃动画
        │
        ▼
8. 重力持续作用，VerticalVelocity 减到 0 后转负，角色下落：
   newState = PlayerAnimState.Fall
        │
        ▼
9. 角色接触地面：
   self.Controller.isGrounded = true
   PlayerViewSystem 检测到落地：
   jumpComp.VerticalVelocity = 0;
   jumpComp.IsGrounded = true;  ← 回写
        │
        ▼
10. PlayerAnimatorSystem 检测到 IsGrounded：
    newState = PlayerAnimState.Idle (或 Move)
    ← 跳跃完成，可再次跳跃
```

## 第9章 常见问题 FAQ

### Q1: 为什么 Player 不动？

**排查步骤**：
1. Console 是否有 `[Entry] Business init finished` 日志？
   - 无：ET.Entry.Start 未被调用，检查 CodeLoader 是否启动成功
2. Console 是否有 `[MainSceneViewInit] Player view injected` 日志？
   - 无：MainSceneViewInit.Init 反射调用失败，检查 Entry.cs 中 Type.GetType 是否返回 null
3. Player 物体是否存在？
   - MainSceneViewInit.CreateSceneContent 动态创建，应该存在
4. CharacterController 是否被正确注入？
   - 检查 viewComp.Controller 是否为 null

### Q2: 为什么报错 "Type or namespace 'UnityEngine' could not be found"？

可能原因：
1. **代码在 Hotfix 层**：Hotfix.asmdef `noEngineReferences: true`，不能 `using UnityEngine;`
   - 解决：把需要 UnityEngine 的代码移到 HotfixView 或 ModelView 层
2. **HotfixView.asmdef 配置错误**：本项目 HotfixView 已改为 `noEngineReferences: false`，如果改回 true 会报错

### Q3: struct 属性修改报错 CS1612？

错误示例：
```csharp
inputComp.Input.JumpRequested = false;  // ❌ Input 是 struct 属性
```

原因：`Input` 是 struct 类型属性，返回的是值副本，不能直接修改其字段。

正确写法（读-改-写回）：
```csharp
InputRequest consumed = inputComp.Input;
consumed.JumpRequested = false;
inputComp.Input = consumed;
```

### Q4: 为什么 Entry.cs 要用反射获取 FiberManager.mainFiber？

因为 `FiberManager.mainFiber` 是 `private` 字段，`Fiber.Instance` 是 `internal static` 属性（[Fiber.cs:23](../../ET/Packages/cn.etetet.core/Scripts/Core/Share/World/Fiber/Fiber.cs#L23)），外部程序集无法直接访问。通过反射绕过访问限制：

```csharp
FieldInfo field = typeof(FiberManager).GetField("mainFiber",
    BindingFlags.NonPublic | BindingFlags.Instance);
Fiber fiber = field?.GetValue(FiberManager.Instance) as Fiber;
Scene scene = fiber?.Root;
```

ET 原版方案：通过 `[Invoke(sceneType)]` 订阅 FiberInit 事件，在事件参数中获取 Fiber，更优雅但复杂。本项目用反射简化。

### Q5: 为什么 JumpSystem 用固定 dt=0.02f 而不是 Time.deltaTime？

因为 JumpSystem 在 Hotfix 层，`noEngineReferences: true` 禁止 `using UnityEngine;`，无法访问 `Time.deltaTime`。

完整 ET 项目中，Hotfix 层用 `TimeInfo.Instance.ClientNow()` 计算时间差，或使用固定步长（帧同步逻辑）。本项目简化为 0.02f（50fps，与 ET 帧同步默认帧率一致）。

实际项目中可改为：
```csharp
long now = TimeInfo.Instance.ClientNow();
float dt = (now - self.LastTickTime) / 1000f;
self.LastTickTime = now;
```

### Q6: 为什么 GameInit 在 Assets/Scripts/ 下而不是业务包里？

因为 GameInit 是 **Unity MonoBehaviour 入口**，需要：
1. 被 Unity 场景中的 GameObject 直接挂载
2. 访问所有 ET 程序集（World / FiberManager / CodeLoader 等）

`Assets/Scripts/` 默认属于 `Assembly-CSharp.dll`，它是 Unity 的默认程序集，**自动引用所有 `autoReferenced: true` 的 asmdef**（包括所有 ET 包）。如果放在业务包里，反而要手动配置 asmdef 引用。

### Q7: 为什么 Model 层代码可以引用 UnityEngine？

因为 `ET.Model.asmdef` 的 `noEngineReferences: false`。

但**完整 ET 项目的 Model 层通常不引用 UnityEngine**，因为 Model 是 C/S 共享的，服务端加载会报错。本项目简化处理，把 `MoveComponent` 的 `Vector3` 字段放 Model 层是**为了学习方便**，实际项目应：
- 服务端也需要的字段 → 用 float / 自定义 struct（不引用 UnityEngine）
- 客户端专属的字段 → 放 ModelView 层

### Q8: 热更新具体是怎么发生的？

本项目已采用 ET 原版 Loader 方案，热更流程：

**编辑器模式**（开发时）：
1. 修改 Hotfix/Share/JumpSystem.cs（如调整 Gravity 值）
2. Unity 自动重编译 ET.Hotfix 程序集
3. 点击 Play，新逻辑立即生效

**运行时模式**（发布后）：
1. 修改 Hotfix 层代码
2. 通过 HybridCLR 编译工具生成新的 `ET.Hotfix.dll`
3. 把 dll 打包成资源包，上传到 CDN
4. 客户端运行时检测到热更版本，下载新 dll
5. 调用 `CodeLoader.Instance.Reload()` 重新加载 Hotfix dll
6. 新 JumpSystem 替换旧 JumpSystem，下次 Update 调用新逻辑

`[EntitySystemOf]` / `[EntitySystem]` 特性由 SourceGenerator 在编译期生成注册代码，新 dll 加载后自动注册到 `EntitySystemSingleton`。

详细原理请参考 [ET Loader说明文档.md](./ET Loader说明文档.md) 第 4.2 节。

## 第10章 扩展方向

学完本项目后，可以按以下方向继续迭代：

### 10.1 增加摄像机跟随

在 ModelView 层新增 `CameraComponent`，在 HotfixView 层新增 `CameraSystem`，每帧让 Camera 跟随 Player Transform。

### 10.2 增加冲刺技能

1. Model 层：在 `InputRequest` 添加 `bool SprintRequested`，新增 `SprintComponent`（含 SprintSpeed 字段）
2. Hotfix 层：`SprintSystem.Update` 处理冲刺状态切换
3. HotfixView 层：`InputSystem.CollectInput` 采集 Shift 键，`MoveSystem.UpdateMove` 根据 SprintComponent 切换 Speed

### 10.3 加入网络同步

1. 在 Proto 中定义 `C2M_MoveRequest` 消息
2. Model 层新增 `MoveSyncComponent`（存储上一次同步位置）
3. Hotfix 层 `MoveSyncSystem` 定时（如 100ms）发送位置到服务端
4. 服务端广播给其他客户端

### 10.4 用 [Invoke] 事件系统替代反射（推荐）

当前 Entry.cs 通过反射调用 MainSceneViewInit，可改为 ET 原版的 `[Invoke]` 事件订阅：

```csharp
// Hotfix/Share/MainFiberInit.cs
[Invoke(1)]  // sceneType=1 对应 GameScene
public class MainFiberInit : AInvokeHandler<FiberInit, ETTask>
{
    public override ETTask Handle(FiberInit args)
    {
        Scene scene = args.Fiber.Root;
        PlayerComponent player = scene.AddComponent<PlayerComponent>();
        // 视图初始化通过 [Event] 事件触发
        return ETTask.CompletedTask;
    }
}
```

这样 Entry.Start 只负责创建主 Fiber，业务初始化由事件系统自动触发，更符合 ET 架构。

### 10.5 服务端权威移动

切到 `ClientServer` 模式，在 Server 目录添加：
- `MoveAuthorityComponent` — 服务端权威位置
- `MoveAuthoritySystem` — 校验客户端移动请求

客户端发送移动请求 → 服务端校验后广播 → 客户端插值平滑。
